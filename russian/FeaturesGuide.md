## Дополнительный гайд 2: Фичи (@features) и группы фич (@feature_groups)

Фичи — это способ разбить мод на независимые включаемые/выключаемые части. Пользователь может выбрать, какие функции мода активировать. Группы фич позволяют объединять связанные функции и задавать тип выбора (один из нескольких или несколько из нескольких).

### @features

В корневом объекте `content.json` можно добавить поле `@features`. Его значение — объект, где ключ — идентификатор фичи (произвольная строка), а значение — объект с настройками фичи.

**Структура фичи:**
```json
"@features": {
  "feature_id": {
    "@name": "Название фичи (обязательно)",
    "@description": "Описание фичи (опционально)",
    "@enabled": true,           // включена по умолчанию? (по умолчанию true)
    "@priority": 0,             // порядок загрузки фичи (по умолчанию: 0, меньше = раньше)
    "@conflicts": ["other_feature"], // список фич, с которыми нельзя включить одновременно
    "@root": "папка_с_ресурсами",    // пока не работает в старых версиях
    "@patches": "путь/к/файлу.json", // внешний файл с патчами (см. гайд 3)
    // дальше могут идти любые патчи таблиц, как в корне мода (например "skins": {...})
  }
}
```

**Пример простой фичи:**
```json
{
  "@title": "Мод с фичами",
  "@features": {
    "extra_skins": {
      "@name": "Дополнительные скины",
      "@description": "Включает набор скинов для Кольта",
      "@enabled": true,
      "skins": {
        "Colt_skin": { "Character": "Colt", "Model": "colt_gold.scw" }
      }
    },
    "custom_music": {
      "@name": "Другая музыка",
      "@enabled": false,
      "music": { "lobby": "mymusic.mp3" }
    }
  }
}
```

### @feature_groups

Группы позволяют объединить несколько фич в один визуальный блок в настройках мода. Можно задать тип группы:
- `DEFAULT` (по умолчанию) — обычные чекбоксы, можно включить несколько фич из группы.
- `RADIO_GROUP` — радиокнопки, можно выбрать только одну фичу из группы.

**Синтаксис:**
```json
"@feature_groups": {
  "group_id": {
    "@name": "Название группы (обязательно)",
    "@description": "Описание группы (опционально)",
    "@type": "RADIO_GROUP", // или "DEFAULT"
    "@features": ["feature_id1", "feature_id2", ...]
  }
}
```

**Важно для RADIO_GROUP:** среди перечисленных фич по умолчанию должно быть включено не более одной (иначе мод не подпишут).

**Полный пример с группой:**
```json
{
  "@title": "Выбор режима",
  "@features": {
    "mode_easy": {
      "@name": "Легкий режим",
      "@enabled": true
    },
    "mode_hard": {
      "@name": "Сложный режим",
      "@enabled": false
    }
  },
  "@feature_groups": {
    "difficulty": {
      "@name": "Сложность",
      "@description": "Выберите уровень сложности",
      "@type": "RADIO_GROUP",
      "@features": ["mode_easy", "mode_hard"]
    }
  }
}
```

### Порядок загрузки мода

1. Сначала загружаются корневые элементы `content.json` (все патчи вне фич). Они всегда активны.
2. Затем последовательно загружаются все включённые фичи. Порядок определяется следующим образом:
   - **Приоритет:** сначала обрабатываются фичи с меньшим значением `@priority`.
   - **ID фичи:** если приоритеты равны, фичи сортируются лексикографически по ID.
   
   Если фича переопределяет какие-то данные, более поздняя фича имеет приоритет.

Это позволяет выносить общие данные в корень, а специфичные — в фичи, не заботясь о зависимостях.

### Ограничения

- Внутри фичи можно использовать `@patches` и `@root` (см. следующий гайд).
- Нельзя одновременно указывать `@patches` и вложенные патчи в теле фичи — только что-то одно.
