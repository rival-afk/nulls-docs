## Дополнительный гайд 1: Мета-информация в модах (название, описание, автор, локализация, HTML)

В файле `content.json` есть три обязательных поля, которые отвечают за отображение информации о моде: `@title`, `@description`, `@author`. Они могут быть простыми строками или поддерживать локализацию и HTML-форматирование.

### Обязательные поля

```json
{
  "@title": "Название мода",
  "@description": "Описание мода",
  "@author": "Автор"
}
```

- `@title` — название модификации. Должно быть кратким и понятным.
- `@description` — описание того, что делает мод. Желательно указывать конкретные изменения.
- `@author` — имя или ник разработчика.

### Локализация (поддержка нескольких языков)

Чтобы мод понимали пользователи из разных стран, можно перевести эти поля на разные языки. Для этого вместо строки используется объект, где ключ — двухбуквенный код языка (ISO 639, заглавными буквами), а значение — текст на этом языке.

Обязательно должен присутствовать ключ `EN` (английский) — он будет использоваться, если язык пользователя отсутствует в словаре.

**Пример:**
```json
{
  "@title": {
    "RU": "Супер мод",
    "EN": "Super mod",
    "TR": "Süper mod"
  },
  "@description": {
    "RU": "Добавляет крутые скины",
    "EN": "Adds cool skins"
  },
  "@author": "Ваш ник"
}
```

Поддерживаются любые языки, которые есть в настройках Android: RU, EN, TR, ZH, ES, DE, FR и т.д.

### HTML-форматирование

Поля `@title`, `@description`, `@author` могут содержать HTML-теги для красивого оформления: жирный текст, ссылки, списки и т.п.

**Доступные теги** (на основе `android.text.Html`):
- `<b>жирный</b>`
- `<i>курсив</i>`
- `<u>подчёркнутый</u>`
- `<s>` или `<del>` — зачёркнутый
- `<font color='#RRGGBB'>цветной текст</font>`
- `<a href='URL'>ссылка</a>`
- `<br>` — перенос строки
- `<ul><li>элемент списка</li></ul>`
- `<p>`, `<div>`, `<span>`, `<cite>`, `<blockquote>`, `<big>`, `<small>`, `<tt>`, `<code>`, `<sup>`, `<sub>`, `<h1>…<h6>`

**Важно:** не забывайте экранировать кавычки внутри JSON.

**Пример с HTML:**
```json
{
  "@title": "<b>Супер мод</b>",
  "@description": "<p>Описание:</p><ul><li><i>Крутая фича</i></li><li><u>Ещё одна</u></li></ul><br>Ссылка: <a href='https://t.me/nb_mods'>Чат</a>",
  "@author": "<a href='https://t.me/danyanull'>данянул</a>"
}
```

### Рекомендации

- Начинайте предложения с большой буквы, ставьте пробелы после знаков препинания.
- Избегайте транслитерации (пишите "default", а не "дефолтный").
- Описание должно быть содержательным: отвечать на вопросы «Что изменится?», «Как это повлияет на игру?».
- Для длинных описаний HTML поможет структурировать информацию.
