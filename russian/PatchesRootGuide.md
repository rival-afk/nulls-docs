## Дополнительный гайд 3: Продвинутая работа с патчами — @patches и @root

Для удобства поддержки больших модов введены два ключа: `@patches` (вынос описаний изменений в отдельный файл) и `@root` (указание папки с ресурсами). Они могут использоваться как на уровне всего мода, так и внутри отдельных фич.

### @patches

Позволяет вынести JSON-объект с патчами таблиц в отдельный файл. Это упрощает `content.json`, особенно если мод содержит много изменений.

**Синтаксис:**
```json
{
  "@patches": "путь/к/файлу.json"
}
```
Путь указывается относительно корня архива мода (не учитывает `@root`).

**Пример:**
Вместо того чтобы писать все патчи внутри `content.json`:
```json
{
  "@title": "Мой мод",
  "skins": { ... },
  "skin_confs": { ... }
}
```

Можно создать файл `patches/skins.json`:
```json
{
  "skins": { ... },
  "skin_confs": { ... }
}
```
И в `content.json` указать:
```json
{
  "@title": "Мой мод",
  "@patches": "patches/skins.json"
}
```

**Важно:** если используется `@patches`, то **запрещено** указывать какие-либо другие патчи (например, `"skins": {...}`) в том же объекте — они будут проигнорированы, мод не подпишут.

`@patches` можно применять как в корне мода, так и внутри фичи. Внутри фичи он работает аналогично.

### @root

Указывает папку внутри архива, в которой лежат все ресурсы мода (текстуры, звуки, модели). Это позволяет не сваливать всё в корень, а организовать структуру.

**Синтаксис:**
```json
{
  "@root": "имя_папки"
}
```

**Пример:**
Если у вас в архиве есть папка `my_res`, а внутри неё стандартные подпапки `sc3d`, `sfx` и т.д., то указав `"@root": "my_res"`, игра будет искать файлы относительно `my_res`, а не корня.

**Важно:** в некоторых версиях (например, 63.286) `@root` мог не работать. Начиная с `@gv:64` он функционирует нормально. Проверяйте совместимость.

`@root` также работает внутри фич: тогда ресурсы из указанной папки будут загружаться только при включённой фиче.

### Совместное использование

Можно комбинировать `@patches` и `@root`. Например:
```json
{
  "@title": "Большой мод",
  "@root": "assets",
  "@features": {
    "feature1": {
      "@name": "Фича 1",
      "@root": "feature1_res",
      "@patches": "patches/feature1.json"
    }
  }
}
```

### Примечание

- Пути в `@patches` всегда относительно корня архива, независимо от `@root`.
- Если вы используете `@root` для фичи, убедитесь, что указанная папка существует в архиве.
