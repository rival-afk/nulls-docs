## Дополнительный гайд 4: UUID и версионирование модов

Правильное использование UUID и версий помогает пользователям автоматически обновлять моды и избегать конфликтов.

### UUID (Modification-Id)

Каждый мод должен иметь уникальный идентификатор в формате UUID. Он указывается в файле `META-INF/MANIFEST.MF` в поле `Modification-Id`.

**Зачем нужен UUID?**
- Если пользователь устанавливает мод с тем же UUID, старый мод автоматически удаляется. Это упрощает обновление.
- UUID используется для идентификации мода в системе и в библиотеке.

**Как сгенерировать UUID?**
- Онлайн-генераторы: https://www.uuidgenerator.net
- Командная строка: `uuidgen` (Linux/macOS) или PowerShell: `[guid]::NewGuid()`

**Правила:**
- UUID должен быть уникальным для каждого нового мода.
- При обновлении мода UUID должен оставаться **тем же самым**.
- Никогда не используйте UUID чужого мода без разрешения автора.

### Версионирование (@version)

Версия мода указывается в `content.json` с ключом `@version`. Формат — строка `X.Y.Z`, где X, Y, Z — целые неотрицательные числа (например, `"1.0.0"`). Если поле отсутствует, по умолчанию считается `"1.0.0"`.

**Смысл чисел:**
- **X (major)** — мажорная версия. Меняется при крупных, концептуальных изменениях.
- **Y (minor)** — минорная версия. Добавление новых функций, не ломающих совместимость.
- **Z (patch)** — патч. Исправление ошибок, мелкие правки.

**Правила обновления версий:**
- При изменении мажорной версии (X) все меньшие (Y, Z) должны быть обнулены: `1.9.5` → `2.0.0`.
- При изменении минорной версии (Y) патч (Z) обнуляется: `1.8.5` → `1.9.0`.
- Патч просто увеличивается: `1.0.0` → `1.0.1`.

**Сравнение версий:**
Загрузчик модов считает более новой ту версию, у которой больше число в старшем разряде. Например, `2.0.0` новее, чем `1.99.99`.

### Использование в подписи и библиотеке

- При ручной подписи (Pro) в заявке обязательно указывать UUID и версию (для обновлений).
- В библиотеке моды сортируются, и пользователь видит самую новую версию.
- Автоматическая подпись через бота генерирует UUID на основе связки `@author` + `@title`, поэтому он будет постоянным для одного и того же мода.

**Пример правильного обновления:**
- Версия 1.0.0: первый релиз.
- Версия 1.1.0: добавлена новая фича.
- Версия 1.1.1: исправлен баг.
- Версия 2.0.0: полностью переработан мод.
