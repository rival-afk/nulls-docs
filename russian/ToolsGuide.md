## Дополнительный гайд 7: Инструменты для мододелов

Для удобства создания и отладки модов сообщество разработало несколько полезных инструментов.

### 1. JSON-схема для модов

Официальная JSON-схема помогает в современных редакторах кода (VS Code, IntelliJ IDEA) подсвечивать ошибки, автодополнять ключи и показывать подсказки.

**Как подключить:**
В самом начале вашего `content.json` добавьте строку:
```json
{
  "$schema": "https://ext.nulls.gg/mods/schema/schema.json",
  ...
}
```

После этого редактор будет проверять структуру файла на соответствие формату модов.

**Где посмотреть документацию по схеме:**  
https://ext.nulls.gg/mods/schema/docs

**Исходный код и предложения:**  
https://github.com/nulls-mods-community/schema

### 2. SC Editor (редактор SC/SC2/SCTX файлов)

SC Editor — Java-приложение для просмотра и редактирования файлов форматов SC (2D-текстуры и анимации), SC2 и SCTX. Позволяет проверять работоспособность изменений и даже рендерить объекты в картинку/видео.

**Где скачать:**  
https://github.com/danila-schelkov/sc-editor/releases/latest

**Возможности:**
- Открытие SC, SC2, SCTX файлов.
- Просмотр содержимого.
- Сохранение в старый формат SC.
- Рендеринг (полезно для контент-мейкеров).

**Запуск:** требуется Java (JRE). Поддерживается Windows, MacOS, Linux.

### 3. Валидация JSON

Перед отправкой мода на подпись всегда проверяйте синтаксис `content.json`. Поможет сайт https://jsonlint.com. Просто вставьте текст и нажмите «Validate JSON».

### 4. TOML-формат (альтернатива JSON)

Бот @nulls_mods_bot теперь принимает файлы в формате TOML (расширение `.toml`). Структура аналогична JSON, но синтаксис более лаконичный.

**Пример content.toml:```
```toml
"@title" = "No Outline"
"@description" = { RU = "Убирает обводку у скинов", EN = "Removes outline from skins" }

[skins."*"]
OutlineShader = "sc3d/shaders/impostor"
```

При подписи бот сам преобразует TOML в JSON внутри `.NullsBrawlAssets`. Удобно для тех, кто не любит лишние скобки и кавычки.

### 5. Генерация UUID
- https://www.uuidgenerator.net
- или команда `uuidgen` в терминале.

### 6. Генерация иконок
Если нет своей иконки, можно сгенерировать на https://icon.kitchen. Получится квадратная PNG нужного размера.

### 7. Проверка ZIP-архива перед отправкой
Убедитесь, что файлы лежат в корне, а не в папке. Например, правильно:
```

mod.zip
├── content.json
├── icon.png
├── sc3d/
└── sfx/

```
Неправильно (лишняя папка):
```

mod.zip
└── mymod/
├── content.json
└── ...

```
