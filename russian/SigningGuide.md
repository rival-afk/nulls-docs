## Дополнительный гайд 5: Подпись модов — автоматическая и ручная (Pro)

Каждый мод для Null's Brawl должен быть подписан, чтобы его можно было установить. Существует два способа подписи: автоматический (через бота) и ручной (через топик #Signing Requests Pro).

### Автоматическая подпись (бот @nulls_mods_bot)

Подходит для простых модов, состоящих из одного JSON-файла (без ресурсов).

**Как подписать:**
1. Отправьте файл `.json` боту в личные сообщения (@nulls_mods_bot) или в топик **#Signing Requests Lite** с командой `/sign` (можно ответом на сообщение с файлом).
2. Через несколько секунд бот вернёт подписанный файл с расширением `.NullsBrawlAssets`.

**Ограничения автоматической подписи:**
- Мод должен состоять только из `content.json` (никаких текстур, звуков, иконок).
- Срок действия подписанного файла — **3 дня**, после чего установить его нельзя.
- Поле `@author` автоматически заполняется вашим ником и Telegram ID (изменять нельзя).
- UUID генерируется автоматически на основе `@author` + `@title` (постоянный для одной связки).
- Иконка (`icon.png`) не поддерживается.

**Когда использовать:** для быстрого тестирования или распространения среди друзей (с учётом срока действия).

### Ручная подпись (Pro)

Подходит для любых модов, особенно тех, которые содержат ресурсы (текстуры, музыку) и предназначены для публикации в библиотеке.

**Процесс:**
1. Подготовьте ZIP-архив, содержащий:
   - `content.json` (обязательно)
   - все необходимые файлы ресурсов (в соответствующих папках: `sc3d`, `sfx`, `music` и т.д.)
   - опционально: `icon.png` (квадратная PNG, не более 640×640)
2. Убедитесь, что файлы лежат в корне архива, а не в промежуточной папке.
3. Сгенерируйте UUID (см. гайд 4) и запишите его. Для нового мода UUID должен быть уникальным, для обновления — совпадать с предыдущим.
4. Отправьте архив в топик **#Signing Requests Pro**.
5. В сообщении обязательно укажите:
   - **Название мода**
   - **Описание** (кратко, что делает мод)
   - **UUID** (в виде `UUID: ...`)
   - **Версию** (для обновлений, например `@version: 1.1.0`)
   - Ссылки на скриншоты/видео (если есть) — они должны вести на сообщения в @nb_mods.
6. Дождитесь, пока волонтёры проверят и подпишут мод. Это может занять от нескольких часов до нескольких дней.

**Требования к моделям для Pro-подписи:**
- Мод должен соответствовать правилам (см. основной гайд).
- Название и описание должны быть осмысленными, без ошибок, желательно с локализацией на русский и английский.
- Иконка (если есть) — только PNG, квадратная, ≤640×640.
- Запрещено использовать чужие UUID без разрешения.
- Если мод обновляется, версия должна быть больше предыдущей.

**Публикация в библиотеку:**
Если вы хотите, чтобы мод попал в библиотеку, добавьте в сообщение фразу «Добавьте в библиотеку, пожалуйста». Мод должен вносить серьёзные изменения (не просто замена одного трека).

### Проверка перед отправкой

- Убедитесь, что `content.json` валидный (можно проверить на https://jsonlint.com).
- Все пути к файлам указаны верно.
- UUID корректен.
- Для модов с фичами проверьте, что нет конфликтов (например, в RADIO_GROUP включено не более одной фичи по умолчанию).
