# Модификации Null’s Brawl

Версия спецификации: unstable (10 января 2026)

### Основные понятия

**Модификация** — специальный пакет, который может быть загружен загрузчиком модификаций.

Изначально представлена в виде директории. ID модификации строго является названием этой директории.

**ID модификации** — корректный UUID, который должен быть уникален для каждой модификации.

### Структура директории

В директории всегда должен содержаться файл **content.json** в формате JSON и содержащий объект [Modification](#Modification).

## Загрузка модификации: патчинг таблиц

Нередко модификациям нужно изменить файлы таблиц, например, skin_confs.csv. Однако такие файлы нельзя менять напрямую — в модификации должны быть описаны **изменения** (дельта),
чтобы загрузчик мог сам «отпатчить» таблицы игры (применить эти изменения). Данный подход выбран с целью минимизировать количество конфликтов между модификациями, а также упростить
обновление модификаций на более новую версию игры.

### Изменения: общее представление

В рамках каждой отдельной таблицы изменения описываются объектом [TablePatch](#TablePatch).

Наглядный пример данного объекта выглядит так:

```json
{
  "PiggyLevel_0_0": {
    "State": 6,
    "ShownLevelInCounter": 5
  }
}
```

Если мы применим описанные изменения к таблице **club_piggy_levels.csv**, то ее первая строка (**PiggyLevel_0_0**) поменяет значения в колонках **State** (на 6) и
**ShownLevelInCounter** (на 5).

Перечень таблиц с изменениями в них описывается объектом [TablePatches](#TablePatches).

Наглядный пример данного объекта для таблицы выше выглядит так:

```json
{
   "club_piggy_levels": {
      "PiggyLevel_0_0": {
         "State": 6,
         "ShownLevelInCounter": 5
      }
   }
}
```

### Изменения: типы значений

Загрузчик ожидает, что указанный в JSON тип значения будет совпадать с типом, который определен в заголовке таблицы.

| Тип в таблице         | Тип в JSON                               |
|-----------------------|------------------------------------------|
| String, StringArray   | Строковой литерал (пример: **"MyText"**) |
| Int, IntArray         | Число без кавычек (пример: **123**)      |
| Boolean, BooleanArray | **true** или **false**                   |

Значение **null** в JSON считается корректным независимо от типа в таблице. Оно позволяет полностью «обнулить» соответствующую ячейку таблицы (что будет аналогично **""** для
текстовых данных).

В случае, если загрузчик определит несовпадение типов данных, он может попытаться исправить это. Например, воспользоваться Integer.parseInt для преобразования строки в число.
Но такое поведение не гарантируется.

### Изменения: массивы в значениях

Допускается использование массивов в значениях.

Это имеет смысл в тех случаях, когда один объект в таблице (в рамках одного Name) может содержать несколько строк.

<details>
<summary>Пример такой таблицы</summary>

| Name      | Colors   | Speed | Scale |
|-----------|----------|-------|-------|
| String    | String   | Int   | Int   |
| FreeOffer | FF9FFF72 | 40    | 100   |
|           | FF68E524 |       |       |
|           | FF68E524 |       |       |
|           | FF9FFF72 |       |       |
|           | FFE0FFA0 |       |       |
| ...       | ...      | ...   | ...   |

В данной таблице объект **FreeOffer** состоит из пяти строк, которые семантически описывают массив значений **Colors**.

В виде JSON это можно описать массивом:

```json
{
  "FreeOffer": {
    "Colors": [
      "FF9FFF72",
      "FF68E524",
      "FF68E524",
      "FF9FFF72",
      "FFE0FFA0"
    ]
  }
}
```

</details>

Если количество строк объекта в таблице (X) отличается от размера массива (Y) в JSON, то поведение загрузчика будет следующим:

1. Y > X (размер увеличился): загрузчик добавит новые CSV строки, чтобы выделить новые ячейки под значения массива;
2. X > Y (размер уменьшился): загрузчик запишет первые Y значений массива, а все последующие клетки ниже будут очищены. Если после очистки клетки остается полностью пустая строка,
   то она будет удалена.

Из информации выше также следуют некоторые свойства:

1. Массив `[X, Y, Z, null]` будет обработан аналогично массиву `[X, Y, Z]`;
2. Массив из одного элемента `[X]` будет обработан аналогично обычному значению `X`.

### Изменения: добавление новых объектов

Если JSON-объект изменений ссылается на неопределенный объект, то он будет создан. Под него будут выделены строки в конце таблицы.

Если таких объектов несколько, точный их порядок в таблице не определен.

## Справочная информация: основные JSON объекты

### Modification

Объект. Описывает модификацию.

| Ключ                                 | Тип                             | Значение                                                  |
|--------------------------------------|---------------------------------|-----------------------------------------------------------|
| @title <sup>(обязателен)</sup>       | [IntlString](#IntlString)       | Название модификации                                      |
| @description <sup>(обязателен)</sup> | [IntlString](#IntlString)       | Описание модификации                                      |
| @author                              | [IntlString](#IntlString)       | Автор модификации                                         |
| @version                             | [Version](#Version)             | Версия модификации; если не указана, используется "1.0.0" |
| @gv <sup>(обязателен)</sup>          | Integer                         | Версия игры, с которой совместима модификация             |
| @ilib <sup>(внутренний)</sup>        | Boolean                         | Опубликована ли модификация в библиотеке                  |
| @categories <sup>(внутренний)</sup>  | String[]                        | Категории для публикации в библиотеке                     |
| @features                            | [Features](#Features)           | Дополнительные (отключаемые) возможности модификации      |
| @feature_groups                      | [FeatureGroups](#FeatureGroups) | Группы дополнительных возможностей                        |
| @root                                | String                          |                                                           |
| @patches                             | String[]                        |                                                           |

Расширяет [TablePatches](#TablePatches) — все прочие ключи относятся к этому объекту.

### Features

Объект с произвольными ключами.

Ключ: текстовый ID возможности.

Значение: объект [Feature](#Feature).

### Feature

Объект. Описывает дополнительные возможности мода.

| Ключ                          | Тип                       | Значение                                                                                            |
|-------------------------------|---------------------------|-----------------------------------------------------------------------------------------------------|
| @name <sup>(обязателен)</sup> | [IntlString](#IntlString) | Название возможности                                                                                |
| @description                  | [IntlString](#IntlString) | Описание возможности                                                                                |
| @enabled                      | Boolean                   | Состояние по умолчанию (включена: если true или ключ отсутствует; выключена: если явно задан false) |
| @conflicts                    | String[]                  | Список из ID возможностей, которые не могут быть включены одновременно с данной                     |
| @root                         | String                    |                                                                                                     |
| @patches                      | String[]                  |                                                                                                     |

Расширяет [TablePatches](#TablePatches) — все прочие ключи относятся к этому объекту.

### FeatureGroups

Объект с произвольными ключами.

Ключ: текстовый ID группы возможностей.

Значение: объект [FeatureGroup](#FeatureGroup).

### FeatureGroup

| Ключ                          | Тип                       | Значение                                                    |
|-------------------------------|---------------------------|-------------------------------------------------------------|
| @name <sup>(обязателен)</sup> | [IntlString](#IntlString) | Название группы возможностей                                |
| @description                  | [IntlString](#IntlString) | Описание группы возможностей                                |
| @type                         | String                    | Тип: DEFAULT или RADIO_GROUP                                |
| @features                     | String[]                  | Список из ID возможностей, которые включены в данную группу |

### IntlString

Локализованная строка.

Значением данного типа может быть:

1. Либо обычная строка;
2. Либо словарь, в котором ключ — код языка, а значение — текстовая строка.

Код языка при этом соответствует стандарту [ISO 639](https://en.wikipedia.org/wiki/List_of_ISO_639_language_codes), однако пишется большими буквами. Например: EN, RU, ZH, FR.

В словаре (в случае его использования) должен присутствовать ключ EN. Он будет использоваться для английского языка, а также для случаев, когда ключа с языком пользователя нет в
словаре.

### Version

Строка в формате "X.Y.Z", где X, Y, Z — числовые значения версии. Например, "1.0.0".

### TablePatches

Объект с произвольными ключами. Описывает перечень таблиц и патчей (изменений) в них.

Ключ: имя файла таблицы без расширения; не может начинаться с @ (пример: skin_confs).

Значение: объект [TablePatch](#TablePatch).

### TablePatch

Объект с произвольными ключами. Описывает изменения в конкретной таблице.

Ключ: название объекта в таблице.

Значение: объект с произвольными ключами (ключ — название колонки, а значение — значения ячеек).

Более подробно о данном объекте написано в разделе [Загрузка модификации: патчинг таблиц](#загрузка-модификации-патчинг-таблиц).
