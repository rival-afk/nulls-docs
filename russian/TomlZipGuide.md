## Дополнительный гайд 5: TOML-формат и подготовка ZIP-архива

### TOML-формат (альтернатива JSON)

Бот @nulls_mods_bot принимает файлы в формате TOML (расширение `.toml`). Структура аналогична JSON, но синтаксис более лаконичный и удобный для чтения.

**Пример content.toml:**
```toml
"@title" = "No Outline"
"@description" = { RU = "Убирает обводку у скинов", EN = "Removes outline from skins" }
"@version" = "1.0.0"

[skins."*"]
OutlineShader = "sc3d/shaders/impostor"
```

**Преимущества TOML:**
- Меньше скобок и кавычек.
- Более читаемая структура.
- Удобно для простых модов.

**Что происходит при подписи:**  
При подписи бот автоматически преобразует TOML в JSON внутри `.NullsBrawlAssets`. Вам не нужно беспокоиться о конвертации.

**Ограничения:**
- Не все возможности JSON доступны в TOML (например, сложные вложенные структуры с массивами могут требовать особого синтаксиса).
- Если вы используете продвинутые функции, возможно, придётся вернуться к JSON.

### Подготовка ZIP-архива перед отправкой

Перед отправкой мода на подпись убедитесь, что архив собран правильно.

**Правильная структура ZIP:**
Файлы должны лежать в корне архива, а не в папке.

Правильно:
```
mod.zip
├── content.json (или content.toml)
├── icon.png
├── sc3d/
└── sfx/
```

Неправильно (лишняя папка):
```
mod.zip
└── mymod/
    ├── content.json
    └── ...
```

**Чек-лист перед отправкой:**
1. `content.json` или `content.toml` находится в корне архива.
2. Файл иконки `icon.png` присутствует (если требуется).
3. Ресурсы (`sc3d/`, `sfx/`, `textures/` и т.д.) лежат в корне или в соответствующих подпапках.
4. В архиве нет лишних папок-обёрток.
5. Архив не повреждён и открывается стандартными средствами.

### Генерация иконок

Если у вас нет своей иконки для мода, можно сгенерировать её на сайте https://icon.kitchen. Выберите подходящий стиль, цвета и размер — вы получите квадратную PNG нужного размера.

**Рекомендуемый размер:** 512x512 пикселей.
